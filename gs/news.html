What's new in mod_lua

Here's a list of new features added to trunk and the estimated release version they will be available in. If you're a fan of mod_lua, you should try the trunk version, or grab the newest release version.

  • Coming in 2.4.10:
    • More verbose error logging


  • New in 2.4.9:
    • The database API now supports returning rows with column names as keys
    • r:parsebody now fully supports uploading binary data
    • Cookies: Fixes a bug where setting multiple cookies are not allowed, also adds support for path values.
    • Cookies: Reworked setcookie entirely, so it now accepts an advanced table of data instead.
    • Using LuaMapHandler alongside FallbackResource will no longer invalidate the map handler when dealing with non-existent file paths


  • New in 2.4.7:
    • WebSocket support: Communicate with browsers in full duplex mode, very fun stuff!
    • Cookie read/write support: Read and write cookies in a secure and orderly manner.
    • Logging hooks: Create your own log files, change data before it gets logged, or skip the logging for specific requests.


  • New in 2.4.6:
    • Multipart form data support: Upload files via mod_lua and more.
    • Input and Output Filters: Filter data going into and out of httpd.
    • Output flushing: Flush data from the output buffer for a real-time display (streaming) of data.
    • Sendfile support: Send entire files using Sendfile, speeding up requests.
    • Server VM scope: A pool of VMs that can be acquired and released, for a good compromise between effiency and RAM usage
    • Code caching options: Set your scripts to be cached...or not
    • LuaMapHandler: Map URIs to scripts and specific functions within


  • New in 2.4.4:
    • Native database support: Use MySQL, PostgreSQL and more within mod_lua!
